Basic Syntax

  • Always…
    • open PHP code using <?php
    • close PHP code using ?>
  • Everything outside <?php & ?> is ignored
  • Separate PHP instructions with ; or ?>
  • Write useful comments
http://php.net/manual/en/language.basic-syntax.php
Commenting 
// This is a one-line c++ style comment
/* This is a multi line comment
   yet another line of comment */
# This is a one-line shell-style comment

Danger

/**
echo 'This is a test'; /* This comment will cause a problem */
*/
http://php.net/manual/en/language.basic-syntax.comments.php

Booleans 
$bool = TRUE;
$bool = FALSE;
  • Simplest type
  • Expresses a truth value
  • Either TRUE or FALSE
  • CaSe-iNsEnSiTiVe (use uppercase)
http://php.net/manual/en/language.types.boolean.php
Integers 
$int = 123; // decimal number
$int = -123; // negative decimal number
$int = 0123; // octal number (83 decimal), 0 precede
$int = 0x1A; // hexadecimal number (26 decimal), 0x precede
$int = 0b101; // hexadecimal number (5 decimal), 0b precede
$int = '1'; // not an integer, but a string
  • Whole number
    • Decimal (base 10)
    • Hexadecimal (base 16)
    • Octal (base 8)
    • Binary (base 2)
  • Positive + or negative -
http://php.net/manual/en/language.types.integer.php
Floating Point Numbers 
$float = 1.234; 
$float = 1.2e3; 
$float = 7E-10;
  • AKA floats, doubles, or real numbers
  • Positive + or negative -
  • Beware of NaN
http://php.net/manual/en/language.types.float.php
Strings 
echo 'this is a simple string';

$str = <<<EOD
Example of "string"
spanning {$multiple} lines
using 'heredoc' syntax.
EOD;
  • Series of characters
  • Can be as large as 2GB
  • Parsed or unparsed
http://php.net/manual/en/language.types.string.php
Unparsed Strings 
'\n' › \n
'\'\$x\'' › '\$x'
'$x' › $x
'{$x[1]}' › {$x[1]}
'{$x["date"]}' › {$x["date"]}
'{$obj->month[12]->name}' › {$obj->month[12]->name}
'{\$x["date"]}' › {\$x["date"]}
'{$x['date']}' › Parse Error
  • Single quoted ' (simplest)
  • Nowdoc syntax (PHP 5.3.0) <<<'EOD' & EOD;
  • Does not interpret special characters
  • Variables are not expanded
Parsed Strings 
"\n" › newline
"\"\$x\"" › "$x"
"$x" › value of x
"{$x[1]}" › value of x[1]
"{$x['date']}" › value of x['date']
"{$obj->month[12]->name}" › December
"{\$x['date']}" › {$x['date']}
"{$x["date"]}" › Parse Error
  • Double quoted "
  • Heredoc syntax <<<EOD & EOD;
  • Interprets special characters
  • Variables are expanded
Arrays
  • Ordered map that associates values to keys
  • AKA list, hash table, dictionary, collection, stack, or queue
  • Array types
    • Simple
    • Associative
    • Multi-dimensional
http://php.net/manual/en/language.types.array.php
Simple Arrays 
$array = array( 10, 20, 30 );
$array = array( 'I', 'am', 'TYPO3', );
$array = [ 'are', 'you?', ]; // as of PHP 5.4
  • Comma-separated Variables
  • Last array element comma is optional
  • PHP 5.4 short array syntax [] vs. array()
http://php.net/manual/en/language.types.array.php
Associative Arrays 
$array = array( 'foo' => 'bar', 'bar' => 'foo', );
$array = array( $key1 => $value1, $key2 => $value2, );
$array = array( 10 => 'foo', 20 => 'bar', ]; // as of PHP 5.4
  • Comma-separated key => value pairs
  • Key is an integer, string or constant
http://php.net/manual/en/language.types.array.php
Multi-dimensional Arrays 
$array = array(
	'fruits'  => array ( 'a' => 'apple', 'b' => 'banana', ),
	'numbers' => array ( 1, 2, 3, 4, 5, 6 ),
	'holes'   => array ( 'first', 5 => 'second', 'third' ),
);

echo $array['fruits']['a']; // apple
echo $array['numbers'][2]; // 3
echo $array['holes'][5]; // second
unset( $array['holes'][0] ); // removes first

$array['apple']['green'] = 'good'; // new multi-dimensional array
  • A simple or associative array of arrays
http://php.net/manual/en/language.types.array.php

Incrementing/Decrementing Operators
  • Doesn't affect booleans
  • Decrementing NULL has no effect
  • Incrementing NULL results in 1

++$a Pre-increment Increments $a by one, then returns $a
$a++ Post-increment Returns $a, then increments $a by one
--$a Pre-decrement Decrements $a by one, then returns $a
$a-- Post-decrement Returns $a, then decrements $a by one
http://de.php.net/manual/en/language.operators.string.php

Control Structures

    • while, do-while

http://php.net/manual/en/language.control-structures.php
if, elseif, else if, else 
if ( expression ) {
	…
} elseif ( expression ) {
	…
} else {
	…
}
  • if, elseif, else if
    • Executes when expression evaluates to Boolean TRUE
  • else
    • Executes when prior if & elseifs evaluate to FALSE
http://www.php.net/manual/en/control-structures.if.php
switch 
switch ($i) {
	case 0:
		echo 'i equals 0';
		break;
	case 1:
	case 2:
		echo 'i equals ' . $i;
		break;
	default:
		echo 'i is not equal to 0, 1 or 2';
		break;
}
  • case - condition to match
  • break - stop case execution
  • default - default condition when no others match
http://www.php.net/manual/en/control-structures.switch.php
while 
$i = 1;
while ($i <= 10) {
	// the printed value would be $i before the increment 
	echo $i++;
}
  • Simplest PHP loop
  • Executes when expression evaluates to Boolean TRUE
  • Doesn't execute if expression is FALSE
http://www.php.net/manual/en/control-structures.while.php
do-while 
$i = 0;
do {
	echo $i;
} while ( $i > 0 );
  • Always executes once
  • Continues executing when expression evaluates to Boolean TRUE
  • Doesn't execute if expression is FALSE
http://www.php.net/manual/en/control-structures.do.while.php
for 
for (expr1; expr2; expr3) { statement }

for ($i = 1; $i <= 10; $i++) { echo $i; }
  • PHP's most complex loop
  • expr1 always evaluated once at loop beginning
  • On each iteration, expr2 is evaluated
  • If expr2 is TRUE, then loop continues
  • If expr2 is FALSE, then loop ends
  • After each iteration, expr3 is executed
http://www.php.net/manual/en/control-structures.for.php
foreach 
foreach (array_expression as $value) { statement }

foreach (array_expression as $key => $value) { statement }
  • Easy way to iterate over arrays and objects
  • Don't call reset( array_expression ) before using
  • When done unset( value )
http://www.php.net/manual/en/control-structures.foreach.php
loop helper - break 
$i = 0;
while (++$i) {
	switch ($i) {
		case 5:
		break 1;  /* Exit only the switch. */
		case 10:
		break 2;  /* Exit the switch and the while. */
…
  • ends execution of the current loop
  • break n - how many nested levels to break out of
  • break 1 is same as break
  • break 0 is invalid
http://www.php.net/manual/en/control-structures.break.php
loop helper - continue 
$i = 0;
while ($i++ < 5) {
	echo "Outer\n";
	while (1) {
		echo "Middle\n";
		while (1) {
			echo "Inner\n";
			continue 3;
		}
		echo "This never gets outputted\n";
	}
	echo "This never gets outputted\n";
}
  • Skips rest of current loop execution
  • continue n - how many nested levels to continue out of
  • continue 1 is same as continue
  • switch can use
http://www.php.net/manual/en/control-structures.continue.php
require, include, require_once, include_once 
include 'vars.php';
include_once 'db.php';
require('http://www.example.com/file.php?foo=1');
require_once 'http://www.example.com/file.txt?bar=2'; // doesn't work
  • Evaluates given file using absolute, relative & URL paths
  • No path? Checks include_path, script & current directory
  • File not found
    • require, require_once - gives fatal error
    • include, include_once - gives warning
  • Use return in include to terminate processing early
http://www.php.net/manual/en/function.include.php
