Terminology 

michael@tlf:~ $
  • Command Line Interface (CLI) – keyboard controlled
  • Graphical User Interface (GUI) – mouse controlled
  • command prompt – place to type a command or program request
  • shell – interacts between you and operating system
  • terminal emulator – program to run shell in
  • virtual terminal – On Linux, an always running shell alternative to GUI
http://www.tuxfiles.org/linuxhelp/shell.html

Which Shell to Use?

  • Worry because it's your working environment
  • Possible benefits
    • Command/filename completion
    • History manipulation
    • Command line editor
    • aliases/functions for custom behaviours
  • Most common shell options…
    • bash (Bourne Again SHell), csh (C-SHell)
    • ksh (Korn SHell), sh, tcsh (Tenex C-SHell), zsh
http://linuxfinances.info/info/unixshells.html

Shell Recommendations

  • Do you work on many different systems? bash
    • Available on all *nix systems
    • Consistent working environment
    • Portable Operating System Interface (POSIX) compliant
  • Do you work only on your own systems? zsh
    • Mostly bash compatible
    • Typo correction
    • Better tab completion
    • Additional helper functions
http://stackoverflow.com/questions/4317247/bash-vs-csh-vs-others-which-is-better-for-application-maintenance

Looking Around the CLI 

michael@tlf:~ $ pwd
/Users/michael

michael@tlf:~ $ ls
Desktop/	Library/	Sites/

michael@tlf:~ $ ls -l
total 4.0K
drwx------+  6 michael staff  204 Aug 13 10:19 Desktop/
drwx------+ 38 michael staff 1.3K Mar 29 18:31 Library/
drwsrws--x+ 33 michael _www  1.1K Aug 12 11:39 Sites/
  • pwd – print working directory
  • ls – list directory contents
  • absolute pathname – complete directory or file path
  • relative pathname – relative directory or file path from current directory
http://www.tuxfiles.org/linuxhelp/linuxfiles.html

Looking at Files 

michael@tlf:~/tmp $ cat hello.txt 
Hello Phnom Penh

michael@tlf:~/tmp $ less lipsum.txt
  • cat – displays the contents of a text file
  • less – like cat, but with pagination
    • " " (space bar) – shows next page
    • b – shows previous page
    • / (forward slash) – searches
    • q – exits
http://www.tuxfiles.org/linuxhelp/examinefiles.html

Moving Around the CLI 

michael@tlf:~ $ pwd
/Users/michael
michael@tlf:~ $ cd ../; pwd

michael@tlf:~ $ cd /Users/michael/Sites/; pwd
/Users/michael/Sites
michael@tlf:~/Sites $ cd; pwd
/Users/michael
  • cd [directory] – change directory
    • cd .. – change directory up
    • no parameters means return home
http://www.tuxfiles.org/linuxhelp/linuxfiles.html

Manipulate Files 

michael@tlf:~/hmm $ ls
date.txt
michael@tlf:~/hmm $ cp date.txt today.txt; ls
date.txt  today.txt
michael@tlf:~/hmm $ mv today.txt tomorrow.txt; ls
date.txt  tomorrow.txt
michael@tlf:~/hmm $ rm tomorrow.txt; ls
date.txt
  • cp file file2 – copy file to file2
  • mv file file2 – rename file to file2
  • rm file – remove file
http://www.tuxfiles.org/linuxhelp/fileman.html

Manipulate Directories 

michael@tlf:~/hmm $ mkdir dir1; ls
dir1/
michael@tlf:~/hmm $ rmdir dir1; ls

michael@tlf:~/hmm $ rm -r dir1; ls

michael@tlf:~/hmm $ cp -r dir1/ dir2; ls
dir1/  dir2/
michael@tlf:~/hmm $ rmdir dir2; ls
dir1/
michael@tlf:~/hmm $ mv dir1/ dir2; ls
dir2/
  • mkdir dir1 – create directory dir1
  • rmdir dir1 – remove empty directory dir1
  • rm -r dir1 – remove non-empty directory dir1
  • cp -r dir1 dir2 – recursively copy directory dir1 to dir2
  • mv dir1 dir2 – rename directory dir1 to dir2
http://www.tuxfiles.org/linuxhelp/dirman.html

Command Line Editors – nano & pico

  • nano & pico – easy to use, inflexible
    • CTRL-X y <enter> – save and exit
    • All commands at the bottom of screen
    • Caret character "^" means Control – ^X means press Ctrl-X

Command Line Editors – vi & vim

  • vi & vim – steep learning curve, flexible, powerful
    • <esc> – Normal mode
      • h to go left, l to go right
      • k to go up, j to go down
    • / (forward slash) – Search mode
      • n find next, N find previous
    • i – Insert mode
      • <esc> :wq <enter> – save and exit
      • <esc> :help <enter> for manual

Regular Expression Basics - Shell Wildcards 

# Move all "linux" named HTML files into directory dir1
michael@tlf:~ $ mv *linux*.html dir1

# List files and directories beginning with mk or git
michael@tlf:~ $ ls {mk,git}*

# List files and directories not starting with letters a to r
michael@tlf:~/bin $ ls [!a-r]*
Wildcard Matches
* zero or more characters
? exactly one character
[abcde] exactly one character listed
[a-e] exactly one character in the given range
[!abcde] any character that is not listed
[!a-e] any character that is not in the given range
{debian,linux} exactly one entire word in the options given
http://www.tuxfiles.org/linuxhelp/wildcards.html

Website Permissions - For TYPO3 Development, The Code 

cd /Users/michael/Sites/aihrus/

# Recursively change owner to michael and group to www
chown -R michael:www *

# Recursively changing file permissions to remove all access, then grant 
# read/write to files & directories, and execute to directories
chmod -R a=,a-s,u+rwX,g+rwX *

# Correct .htaccess permissions to be readable
chmod a=,a-s,u+rw,g+rw .htaccess

# Enable cron file execution
find . -type f \( -name "*.sh" -o -name "*.phpsh" \) -exec chmod ug+x {} \;

# Enabling SETUID and SETGID on directories - clean ownership
find * -type d ! -type l -exec chmod ug+s {} \;
chmod ug+s /Users/michael/Sites/aihrus/

# Enabling exec on /Users/michael/Sites/aihrus/
chmod a+X /Users/michael/Sites/aihrus/
